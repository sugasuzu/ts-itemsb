# 全通貨分析レポート

**実行日:** 2025-11-10
**対象通貨:** 11通貨（AAVE, ADA, ALGO, ATOM, AVAX, BNB, BTC, DOGE, DOT, ETC, ETH）
**総ルール数:** 26,200個

---

## エグゼクティブサマリー

### 主要な発見

1. **ETC（Ethereum Classic）が圧倒的に最多**
   - 発見ルール数: 6,651個（全体の25.4%）
   - 高サポートルール: 1,091個
   - 総合スコア: 69.6点（1位）

2. **統計的有意性ではDOGEが最強**
   - 最高|mean|: **0.965%**（目標の0.5%を大幅超過）
   - |mean|≥0.5%のルール: 56個

3. **BTCは意外にも低パフォーマンス**
   - 総合スコア: 9.2点（**11位/11通貨**）
   - |mean|≥0.5%のルール: **0個**
   - 最高|mean|: 0.479%（目標未達）

4. **全体的な問題: σ制約が緩すぎる**
   - σ<0.3%のルール: **全26,200個中わずか1個**
   - 平均σ: 0.73%（目標の0.5%より大きい）

## 重要な発見と考察

### 🔴 発見1: BTCのパフォーマンスが最低レベル

**事実:**
- 総合スコア: **11位/11通貨**
- |mean|≥0.5%のルール: **0個**
- 高サポートルール: 99個（最少）

**仮説:**

1. **BTCの市場効率性が高い**
   - BTCは最も取引量が多く、価格発見が効率的
   - 統計的異常パターンが発生しにくい
   - ⇒ ランダムウォークに近い

2. **属性の相関が弱い**
   - BTCは独立した動きをすることが多い
   - 他の仮想通貨との相関パターンが少ない

3. **現在の属性セットがBTCに不適**
   - 20通貨×3状態の属性はアルトコイン中心の関係性を捉えている
   - BTC独自の特徴（例: BTC dominance）が含まれていない

**推奨:**
- BTCには別の属性セット（マクロ指標、BTC独自の指標）が必要かもしれない
- あるいは、BTCの「効率的市場」という特性自体が興味深い研究結果

---

### 🟢 発見2: DOGEが統計的異常の発見で最強

**事実:**
- 最高|mean|: **0.965%**（全通貨中1位）
- |mean|≥0.5%のルール: 56個（3位）

**解釈:**
```
DOGEの統計的特性:
  μ_data ≈ 0% (仮定)
  σ_data ≈ 0.5% (仮定)

  最高ルール: |mean| = 0.965% ≈ μ + 1.9σ
  → p値 < 0.03（5%水準で有意）
```

**仮説:**

1. **ボラティリティが高い**
   - DOGEは価格変動が激しい
   - 極端なパターンが発生しやすい

2. **センチメント駆動**
   - ソーシャルメディアの影響を強く受ける
   - 特定の条件下で一方向に動きやすい

3. **流動性の問題**
   - 取引量がBTCより少ない
   - 大口の動きで価格が大きく変動

**研究価値:**
- DOGEの0.965%ルールは論文の目玉になる
- 「センチメント駆動型通貨の統計的異常検出」というテーマ

---

### 🟡 発見3: ETCが量・質ともに優秀

**事実:**
- ルール数: 6,651個（1位）
- 高サポートルール: 1,091個（1位）
- 総合スコア: 69.6点（1位）

**解釈:**
- ETCは「発見しやすい」通貨
- パターンが多様で、サポート率も高い

**仮説:**

1. **中程度の流動性**
   - BTCほど効率的でなく、ALGOほど流動性が低くない
   - パターンが発生しやすい「甘いゾーン」

2. **他通貨との相関が強い**
   - ETHのフォークコインとしての特性
   - 他のPoW通貨との連動性

**実用性:**
- ETCは実トレード戦略への応用可能性が最も高い

---

### 🔵 発見4: σ制約が全体的に緩すぎる

**現状:**
```
Maxsigx = 1.0%
σ<0.3%のルール: 1個 / 26,200個 = 0.004%
平均σ: 0.73%
```

**問題:**
- 適応度関数で分散を評価しているが、フィルタが緩すぎる
- 結果的に高分散ルールが多数発見されている

**推奨対応:**
```c
// 現在
#define Maxsigx 1.0

// 推奨
#define Maxsigx 0.5  // または 0.4
```

**期待される効果:**
- ルール数: 26,200個 → 10,000-15,000個
- 平均σ: 0.73% → 0.45%
- 低σルール数: 1個 → 500-1,000個

## 適応度関数への示唆

### 現在の問題点の裏付け

1. **σ制約が緩すぎる**
   - 証拠: σ<0.3%のルールが全26,200個中1個のみ
   - 対応: `Maxsigx 1.0 → 0.5`

2. **サポート率の評価が不十分**
   - 証拠: 高サポートルール（≥50マッチ）が3,955個も存在
   - 対応: `FITNESS_SUPPORT_WEIGHT 3 → 400`

3. **統計的有意性の評価が機能している**
   - 証拠: 10/11通貨で|mean|≥0.5%のルールが発見されている
   - 結論: `STATISTICAL_SIGNIFICANCE_BONUS`の設計は適切
